# Pizza Yolo Website Wireframe

---

# Purpose

This document defines the complete structure, hierarchy and user journey of the Pizza Yolo website.

This is **not** a visual design document.

It defines:

- page hierarchy
- section order
- information architecture
- user flow
- conversion points
- UX priorities

The visual styling is documented separately in `design-system.md`.

---

# Design Principles

The website should feel:

- fast
- spacious
- modern
- premium
- effortless

Every section should have one clear purpose.

Avoid unnecessary content.

The website should be scrollable in approximately 60–90 seconds.

Large photography should replace large paragraphs whenever possible.

---

# Mobile Experience

Mobile First.

Everything should feel designed specifically for phones.

Large buttons.

Large typography.

Minimal text.

Thumb-friendly spacing.

Sticky CTA.

Fast loading.

---

# User Experience Rules

Never force users to read.

Use visuals whenever possible.

Never hide important information.

Menu should never be more than one click away.

Location should always be accessible.

# Success Criteria

After 30 seconds, a visitor should know:

✓ What Pizza Yolo is

✓ What makes it different

✓ Where it is

✓ What they can order

✓ Why they should visit

Without reading long paragraphs.

If a section does not help answer one of those questions, it should not exist.
